# Формат расчёта пула кубов (компактный)

Краткая строка: список применимых источников + итого + сложность. Без таблиц, без заголовков, без пояснений.

## Шаблон
```
🎲 Пул (X кубов): <черта A> +1, <аспект 1> +1, <черта B> +1, <аспект 2> +1, флаг «<F>» +1. Резерв?
Сложность: N.
```

## Пример
```
🎲 Пул (5 кубов): Кузнечное чутьё +1, чувство скрытых механизмов +1, Чужак в магии +1, скептицизм к иллюзиям +1, флаг «Не доверяет магии» +1. Резерв?
Сложность: 2.
```

## Правила
- Одна строка перечисления, запятые. Каждый элемент = «имя +1».
- НЕ указывать из какой черты аспект (лишний шум — игрок помнит лист).
- НЕ рисовать таблицу с колонками «Источник | Обоснование | Кубы» — это пустая трата токенов.
- «Резерв?» в конце — короткий вопрос игроку, сколько куб кинуть из резерва.
- После ответа игрока пул дополняется резервом и только затем идёт бросок.

## Если строка вылезает за комфортный размер
Только если применимых источников действительно много (6+) и одна строка становится нечитаемой — оборачивать в Discord код-блок:
```
🎲 Пул (7 кубов):
  Черта A +1
  Аспект 1 +1
  Черта B +1
  Аспект 2 +1
  Флаг «F» +1
  Аспект 3 +1
  Черта C +1
Сложность: 3. Запас: 4/7. Сколько из резерва?
```

Это исключение, а не правило — в 90% случаев одна строка лучше.

## Запрещено
- Таблица с колонками «Подходит / Возможно / Не подходит» — это РАЗБОР, который идёт когда игрок явно спрашивает «какие у меня возможности?». В обычной заявке — просто пул и всё.
- Markdown-таблица (`| Source | Reason | Dice |`) БЕЗ оборачивания в код-блок — Discord не рендерит таблицы и это превращается в нечитаемую мешанину.
