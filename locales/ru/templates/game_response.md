# Шаблон ответа в игровой канал

Стандартный скелет ответа ГМ в Discord (игровой канал). Каждый ход обязан включать: (1) то, на что игрок должен отреагировать (решение / бросок / выбор), (2) счётчик запаса кубов если он изменился.

Полный нарратив (длинные описания) идёт в отдельный канал через webhook — game_channel получает КРАТКУЮ версию + механику + call-to-action.

## Тип 1. Действие → объявление броска

```
<1-2 строки: переход от прошлого момента, если это нужно игроку чтобы понять ситуацию>

🎲 Пул (N кубов): <черта A> +1, <аспект> +1, <черта B> +1, флаг «<F>» +1.
Сложность: X. Запас: <current>/7.
Сколько из резерва? (или «нет»)
```

## Тип 2. Бросок выполнен → результат

```
<вывод roll.py целиком>

<1-3 строки интерпретации: что получилось с точки зрения механики>

<если игрок нарратит — предложение с учётом уровня прав (см. rules/SKILL.md §11b):
 «Да, и… — ты рассказываешь. Хочешь +1 куб на следующий бросок, −1 сложности на одну проверку, или снять одно состояние?»>

Запас: <new>/7. <состояния если изменились>
```

## Тип 3. Авто-успех (короткое действие)

```
<1-3 строки компактного нарратива в стиле игры>

<если есть выбор для игрока — call-to-action: «Что дальше?» или конкретный вопрос>
```

## Тип 4. Описание сцены / ответ на вопрос

```
<описание сцены / ответ>

<никакого механического блока, если бросок не нужен>
```

## Обязательные правила для ЛЮБОГО ответа

- **Всегда** есть call-to-action или понятный игроку следующий шаг — кроме тех случаев когда явно ждём только нарратив игрока
- **Всегда** показывать запас кубов если он изменился в этом ходе
- **Не вываливать** полный нарратив в game channel — он идёт в webhook
- **Не делать** анализ всех черт/аспектов списком — только применимые к конкретному действию
- **Многострочные структуры** (таблицы, массивы кубов, списки состояний) — оборачивать в ```кодовый блок``` для читаемости в Discord
- **Один код-блок** на один связный механический вывод (а не 5 подряд)

## Что НЕ нужно писать

- Полную прозу нарратива (она в webhook)
- Обоснование почему каждая черта игрока подходит или нет (если игрок не спрашивает прямо)
- Статусы вида «файлы обновлены», «нарратив отправлен» — это тех.детали, игроку они не нужны
- Повторение сцены которую игрок только что прочитал в webhook
