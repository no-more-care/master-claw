# MasterClaw — Архитектура

## Обзор

MasterClaw — AI-гейммастер для настольной RPG BlackBirdPie, работающий на microClaw (Claude как агент). Два интерфейса: Discord (игроки) и Telegram (оператор).

## Три слоя системы

### 1. Souls — Личности
Контекстные промпты, определяющие поведение AI.

| Soul | Файл | Канал | Назначение |
|------|-------|-------|------------|
| GameMaster | `souls/gamemaster.md` | Discord | Ведение игры для игроков |
| Operator | `souls/operator.md` | Telegram | Управление мирами, диагностика |

### 2. Skills — Процедуры
Модульные пошаговые инструкции для конкретных механик.

| Skill | Файл | Когда используется |
|-------|-------|--------------------|
| rules | `skills/rules/SKILL.md` | Авторитетный источник правил (загружается 1 раз) |
| actions | `skills/actions/SKILL.md` | Обработка заявок игроков (броски, результаты) |
| characters | `skills/characters/SKILL.md` | Создание/изменение персонажей |
| narrator | `skills/narrator/SKILL.md` | Описания мира, диалоги NPC |
| session | `skills/session/SKILL.md` | Старт/продолжение/завершение игр |
| world | `skills/world/SKILL.md` | Генерация событий и реакций мира |
| worldgen | `skills/worldgen/SKILL.md` | Создание новых миров из описания |

### 3. Working Directory — Данные
Персистентное хранилище: миры, игры, персонажи, логи.

```
working_dir/shared/GameMaster/
├── worlds/<world_name>/
│   ├── world.md          — Описание мира, локации, фракции
│   ├── npcs.md           — Карточки NPC
│   ├── plot.md           — Сюжет (СЕКРЕТНО, не для игроков)
│   ├── player_guide.md   — Что знают персонажи
│   └── starter_characters.md — Готовые персонажи (опционально)
└── games/<game_name>/
    ├── game.md           — Метаданные игры (статус, игроки)
    ├── state.md          — Текущее состояние (ПЕРЕОПРЕДЕЛЯЕТ worlds/)
    ├── log.md            — Хронологический лог событий
    └── characters/
        └── <name>.md     — Листы персонажей (YAML-схема)
```

## Режимы работы GameMaster

```
MODE 1: OPERATOR WORK ─── Нет активной игры
  │  Skills: worldgen, session
  ▼
MODE 2: PREPARATION ──── Игра создана, до старта
  │  Skills: characters, narrator
  ▼
MODE 3: GAME ──────────── Активная сессия
  │  Skills: actions, narrator, world, characters
  ▼
MODE 1: Игра сохранена, ожидание продолжения
```

## Поток данных

1. **Оператор** (Telegram) → `session` создаёт папку игры
2. **Игрок** (Discord) заявляет действие → `actions` обрабатывает через лист персонажа
3. ГМ нужен контекст → `narrator` читает world.md + npcs.md + state.md
4. Мир реагирует → `world` генерирует событие, обновляет state.md + log.md
5. Кампания растёт → файлы персонажей накапливают опыт, состояния, аспекты

## Иерархия авторитетности данных

1. `skills/rules/SKILL.md` — единственный источник правил
2. `state.md` переопределяет `worlds/` (мёртвый NPC остаётся мёртвым)
3. Лист персонажа — закон (нет в листе = нельзя использовать)
4. Подтверждение игрока обязательно перед броском
